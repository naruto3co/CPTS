Sử dụng cả option bypass
  --no-cast để dò quét lỗ hổng sql injection. Nó có thể bypass được lớp phòng thủ WAF 
  --prefix='…' trong sqlmap dùng để chèn thêm một đoạn ký tự đứng trước payload SQL injection, giúp bypass filter hoặc vá lỗi cú pháp trước vị trí inject. 
               --prefix='`'
                      Tại sao lại cần thêm dấu ` ?
                          Bởi vì:
                          Một số hệ thống yêu cầu đóng chuỗi, đóng identifier bằng backtick.
                          Một số query đang bị “hở” dấu \` → thêm vào để đóng syntax. 
                          Một số filter/WAF chặn payload bình thường, nhưng lại cho qua nếu có backtick trước.
                          Một số ứng dụng hay viết query kiểu:
                              SELECT * FROM users WHERE name=\`<input>\`
             Hay dùng --prefix="%'))"
                       --suffix="-- -"
                       --prefix='`'   ( chú ý kí tự nhé )

    --union-col=5

| Tham số     | Giá trị          | Ý nghĩa chi tiết                                                             | Ảnh hưởng khi chạy                                                               |
| ----------- | ---------------- | ---------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| **--level** | 1–5              | Xác định mức độ sâu của các bài kiểm tra mà sqlmap thực hiện.                | Level càng cao → sqlmap gửi nhiều request hơn và test thêm nhiều điểm injection. |
|             | **1 (mặc định)** | Chỉ test các tham số cơ bản (GET, POST).                                     | Nhanh, ít chính xác.                                                             |
|             | **2**            | Test thêm Cookie, User-Agent, Referer.                                       | Tìm được nhiều SQLi hơn.                                                         |
|             | **3**            | Test thêm nhiều header và tham số ẩn.                                        | Hữu ích cho SQLi trong cookie.                                                   |
|             | 4–5              | Test rất sâu, brute force header, cookies phức tạp.                          | Chậm nhưng quét rất kỹ.                                                          |
| **--risk**  | 1–3              | Mức độ “mạnh” của payload mà sqlmap dùng, tức nguy cơ gây lỗi trên ứng dụng. | Risk càng cao → payload càng mạnh → dễ tìm lỗi hơn.                              |
|             | **1 (mặc định)** | Payload an toàn, chỉ đọc dữ liệu.                                            | Ít rủi ro, độ chính xác trung bình.                                              |
|             | **2**            | Payload mạnh hơn, có thể gây lỗi nhỏ hoặc chậm trang.                        | Tìm thêm kênh SQLi ẩn.                                                           |
|             | 3                | Payload rất mạnh, có thể làm hỏng dữ liệu (không phù hợp sản phẩm thật).     | Dùng cho lab, pentest sâu.                                                       |
| **--batch** | Không có giá trị | Tự động chọn câu trả lời mặc định khi sqlmap hỏi. Không có prompt yes/no.    | Chạy nhanh, không cần thao tác thủ công.                                         |


| Tham số                                                                     | Ý nghĩa                                  | Công dụng                    | Khi nào dùng                                   |
| --------------------------------------------------------------------------- | ---------------------------------------- | ---------------------------- | ---------------------------------------------- |
| **--tamper=NAME**                                                           | Dùng script thay đổi payload             | Bypass firewall, WAF, filter | Khi web chặn `'`, `"`, `--`, `OR`, `UNION`, vv |
| Ví dụ: `--tamper=space2comment`                                             | Thay khoảng trắng bằng comment `/\*…\*/` | Lách rule chặn space         | WAF chặn khoảng trắng                          |
| `--tamper=between`                                                          | Biến đổi keyword                         | Bypass từ khóa bị chặn       | Web block “AND”, “OR”                          |
| `--tamper=charunicodeescape`                                                | Encode ký tự thành Unicode               | Bypass filter ASCII          | Web scan ASCII payload                         |
| **Ghi nhớ:** bạn có thể dùng nhiều script: `--tamper=space2comment,between` |                                          |                              |                                                |

| Tham số                | Ký tự | Kỹ thuật            | Đặc điểm              | Khi nào dùng                                   |
| ---------------------- | ----- | ------------------- | --------------------- | ---------------------------------------------- |
| **--technique=BEUSTQ** | B     | Boolean-based blind | An toàn, chậm         | Không thấy lỗi nhưng trang phản hồi true/false |
|                        | E     | Error-based         | Lấy dữ liệu rất nhanh | Trang hiển thị lỗi SQL                         |
|                        | U     | Union-based         | Cực nhanh             | Cho phép UNION SELECT                          |
|                        | S     | Stacked queries     | Dùng nhiều câu lệnh   | DB cho phép `;` → RCE                          |
|                        | T     | Time-based blind    | Chậm nhất             | True/False nhưng không thay đổi giao diện      |
|                        | Q     | Inline queries      | Dùng subquery         | Một số DB đặc biệt                             |

Ví dụ: Chỉ test boolean + union 
  --technique=BU

Enumeration mức Database > Table > Column
| Option                 | Chức năng               | Ý nghĩa                           |
| ---------------------- | ----------------------- | --------------------------------- |
| `--dbs`                | Liệt kê tất cả database | Xem app dùng DB nào               |
| `-D <db>`              | Chọn database           | Dùng kèm tables/columns/dump      |
| `--tables`             | Liệt kê bảng trong DB   | Xác định bảng có dữ liệu nhạy cảm |
| `-T <table>`           | Chọn bảng để tương tác  | Chuẩn bị dump                     |
| `--columns`            | Liệt kê cột của bảng    | Tìm cột password/email/hash       |
| `-C <column1,column2>` | Chọn cột cần dump       | Dump gọn, nhanh, tránh noise      |


Dump dữ liệu
| Option                          | Chức năng                  | Giải thích                            |
| ------------------------------- | -------------------------- | ------------------------------------- |
| `--dump`                        | Dump dữ liệu từ bảng       | Xuất data ra console + file           |
| `--dump-all`                    | Dump toàn bộ DB server     | Thường dùng khi có full quyền         |
| `--start=N`                     | Dump bắt đầu từ dòng N     | Hữu ích với bảng lớn                  |
| `--stop=N`                      | Dump đến dòng N            | Nhắm mục tiêu dữ liệu                 |

Những option hỗ trợ kỹ thuật Enumeration
| Option                    | Chức năng                   | Ghi chú                       |
| ------------------------- | --------------------------- | ----------------------------- |
| `--batch`                 | Tự động chọn default answer | Chạy mượt, không bị hỏi       |
| `--tamper=…`              | Bypass WAF                  | Nên dùng khi fail payload     |
| `--threads=N`             | Tăng tốc                    | Nhưng đôi khi gây lỗi         |
| `--no-cast`               | Loại bỏ CAST() khỏi payload | Bypass WAF/filter             |
| `--prefix='…'`            | Thêm ký tự trước payload    | Fix syntax hoặc bypass filter |
| `--suffix='…'`            | Thêm ký tự sau payload      | Đóng chuỗi/câu lệnh           |
| `--union-cols=N`          | Chỉ định số cột UNION       | Tăng tốc UNION-based          |
| `--technique=U/E/B/S/Q/T` | Chọn kỹ thuật SQLi          | U=Union, B=Blind…             |
